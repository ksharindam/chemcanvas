3. Import Image via Optical Structure Recognition (OSR)

What it does:
Converts raster images of chemical structures (scans, screenshots, PDFs) into editable molecular objects.

Detailed behavior:
The system accepts image files (PNG, JPEG, TIFF, BMP, GIF) and passes them to an OSR engine. The engine detects bonds, atoms, charges, stereochemistry, and text labels. The resulting structure is rendered on canvas for verification and editing.

Expected behavior:
The recognized structure should closely match the original image. Recognition errors should be visually flagged. Users should be able to edit the result immediately.

Official basis:
ChemAxon integrates OSR engines (CLiDE, OSRA, Imago) for image-to-structure conversion.

4. CLiDE Image Recognition Engine

What it does:
Uses the CLiDE (Chemical Literature Data Extraction) engine for structure recognition from images.

Detailed behavior:
CLiDE analyzes pixel patterns to identify bonds, atom labels, and stereochemical wedges. It handles various drawing styles, including hand-drawn sketches and publication-quality diagrams. The engine returns a confidence score.

Expected behavior:
High-quality images should yield >95% accuracy. The system should handle fused rings, stereochemistry, and common abbreviations.

Official basis:
CLiDE integration is documented in Marvin's image import and OSR modules.

5. Imago Image Recognition Engine

What it does:
Uses the Imago engine for image-to-structure conversion.

Detailed behavior:
Imago processes raster images using machine learning and rule-based pattern recognition. It segments the image into chemical components and reconstructs the molecular graph with bond orders and atom types.

Expected behavior:
Should handle low-resolution images and various font styles for atom labels. Output should be standard Marvin structure objects.

Official basis:
Imago is listed among supported OSR engines in Marvin documentation.

6. OSRA Image Recognition Engine

What it does:
Uses OSRA (Optical Structure Recognition Application) for converting images to structures.

Detailed behavior:
OSRA is an open-source engine that processes images through segmentation, vectorization, and graph reconstruction. Marvin can route images to OSRA as an alternative recognition backend.

Expected behavior:
OSRA should produce valid MOLfiles or SMILES that Marvin can render. It should handle dashed wedges, charges, and isotopic labels.

Official basis:
OSRA integration is documented in Marvin's image import settings.

Reverses the most recent editing operation.

Detailed behavior:
Marvin maintains a multi-level undo stack recording every structural change, style change, and object manipulation. Undo restores the previous state without losing the redo possibility.

Expected behavior:
Undo should revert exactly one operation per invocation. The stack should be deep enough for complex drawing sessions (typically 50+ levels).

Official basis:
Multi-level undo is standard in Marvin's editing system.
